My partner is Jordan Hendley; we partnered with Lance Petersen and Christian Gallegos for the schema review.

We decided on 4 tables: Users, Students, Form, and Committee. The Users table contains a unique ID, First Name, Last Name, and position (faculty or student). The Students tabel contains a foreign key from Users for the ID, First Name, Last Name, Advisor Last Name, Track, Degree, Sem. Admitted, and Committee members. The Committee table contains the Form ID, Student ID and Advisor Name. Finally, the Form table replicates the Due Progress Form, with all of it's included fields.

There were no changes to the schema after meeting with the other group.
