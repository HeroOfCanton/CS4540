The School of Computing tracks Graduate student progress each semester to make sure the students are making satisfactory progress toward their degree. We will be implementing a system to move this manual/paper-based process into the ether, by showing some static proof of concept web pages that will (a) allow the display of a list of students, (b) allow the list of multiple progress reports per student, and (c) show the status of a student's specific progress report.

I wanted the UI choices to be simple and clean. I completely redid the styling of my site, trying to avoid the image and transparent tables from last time. I added some custom buttons, with animation, instead of the tables I was using. I also added a gradient to the background

I have tried to do some of the things mentioned for superior work. If a form doesn't save properly, there is messaging. Also, the form doesn't wipe itself clean when, rather the correct information is retained for the next submission. I have a sign feature, which you can see here: this form shows where the 'Sign Form' button goes; the button only appears when the form is unsigned, and this advisor shows what you would see, as an advisor, if you have forms that are unsigned. All grad forms submitted are unsigned, and will appear in the Advisors view until signed.

The double verification of the form input data comes from the front end, where I have regex patterns, and fields are required, and the back end, where I check to see if the various $_POST variables are empty, and if so, display an error. While most browsers support the 'required' form tag, Safari doesn't, which would allow a user to submit and empty form. However, the checking of $_POST prevents that from actually being added to the database.

Testing strategy was simple: Use OSX, Window 7, Windows 10, and Ubuntu Linux, along with their native browsers and extra browsers, like Chrome, and Firefox, to make sure the page renders correclty. Only Support for IE 11 and up, as Microsoft has terminated support for IE9 and 10.
